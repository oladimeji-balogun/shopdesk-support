## overview
An end-to-end, production-grade autonomous AI support agent that routes, reasons, and responds powered by RAG, tool calling, and intent classification.

Every user query passes through a Router Agent that classifies intent into one of three paths knowledge base lookup, live database query, or human escalation. 
The Orchestrator coordinates the appropriate worker: a RAG chain for policy questions, a tool-calling chain for order and account data, or an escalation handler that queues the issue for a human agent. 
The final response is generated by an LLM with full conversation history for context.

## features 
- **Intent-based routing** -- classifies every query into RAG, tool call, or escalation before any processing happens - ensuring the right handler is always invoked
- **RAG pipeline** -- semantic similarity search over the knowledge base using fastembed embeddings for fast, accurate retrieval
- **Accurate tool calling** -- for performing complex database lookups for information
- **Super fast document processing** -- effecient parsing and processing of text documents making it available for semantic search from the RAG pipeline
- **Conversation memory** -- for persistence and carrying forward the conversation context between user and the agent
- **JWT authentication** -- authorizing each user to prevent unauthorized access with JWT access and refresh tokens
- **Structured logging** -- a structured logging system for dbugging and tracking
- **Rate limiting** -- prevents abuse of backend endpoints using slowapi
- **Docker containerization** -- containerize the whole system into a portable unit which can be run on any machine


## Tech Stack

| Category | Technology | Purpose |
|----------|------------|---------|
| Backend | FastAPI | REST API framework for handling all incoming requests and routing |
| LLM Provider | Groq | Ultra-fast LLM inference for intent classification and response generation |
| AI Framework | LangChain | Orchestrates RAG chains, tool calling, and prompt management |
| Vector Database | Pinecone | Stores and queries text embeddings for semantic knowledge base retrieval |
| Embeddings | fastembed | Encodes text into semantic vectors for similarity search |
| Relational Database | PostgreSQL | Persists sessions, messages, orders, accounts, and escalation tickets |
| Authentication | JWT + bcrypt | Stateless auth with signed access tokens and secure password hashing |
| Containerization | Docker | Packages the full system into portable, reproducible containers |
| Rate Limiting | slowapi | Protects endpoints from abuse with per-IP request throttling |
| Logging | Python logging | Structured rotating file and console logs for observability and debugging |
| Package Management | Poetry | Dependency management and virtual environment isolation |
| Data Validation | Pydantic | Enforces strict type safety on all API request and response models |


## Architecture

![architecture-diagram](/assets/architecture-diagram.jpg)

Every incoming request hits the FastAPI layer which handles authentication and rate limiting before passing the query to the Orchestrator. 
The Router Agent powered by a fast Groq LLM with structured output classifies the intent into one of three paths. 
RAG queries are answered using semantic search over a Pinecone knowledge base. 
Tool call queries trigger live Postgres lookups for order and account data. Escalations create a ticket in the queue for human follow-up. 
All conversation turns are persisted in Postgres for context across sessions.

## Getting started

1. Prerequisites: 
    - download and install: [python](https://www.python.org/downloads/release/python-31213/)
    - download and install: [docker](https://docs.docker.com/desktop/setup/install/windows-install/)
    - install: [poetry](https://python-poetry.org/docs/)

2. Installation: 
    ```bash
    git clone https://github.com/oladimeji-balogun/shopdesk-support
    ```
    ```bash
    cd shopdesk-support 
    ```
    ```bash
    poetry install 
    ```
3. Environment variables: 
    - create a .env file in the project root
    - copy the content of the .env.example file into your .env file 
    - fill in all the the fields as approapriate 
4. Run locally: 
    ```bash
    poetry run alembic upgrade head
    ```

    ```bash
    poetry run uvicorn main:app --reload 
    ```

5. Run with docker: 
    ```bash
    docker-compose up --build 
    ```

## API documentation

### Authentication
| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| POST | `/auth/register` | No | Create a new user account and receive access and refresh tokens |
| POST | `/auth/login` | No | Login with email and password and receive access and refresh tokens |
| POST | `/auth/refresh` | No | Exchange a refresh token for a new access token |
| POST | `/auth/logout` | No | Logout the current user |

### Sessions
| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| POST | `/sessions` | Yes | Create a new conversation session for the authenticated user |

### Chat
| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| POST | `/chat/{session_id}` | Yes | Send a message and receive an AI-generated response |

### Queue
| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| GET | `/queue` | Yes | Retrieve all open escalation tickets for human agents |

### Health
| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| GET | `/health` | No | Check the health status of the API and database connection |
